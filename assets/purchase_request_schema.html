Solution Architecture
How Power Platform layers work together to deliver the purchase request system
Overview: The solution centralises procurement governance and replaces email-based approval processes with a structured, auditable workflow. Users create purchase requests, add line items and supporting attachments, and submit requests into a governed approval process. Dataverse serves as the transactional data platform. Power Automate evaluates approval requirements based on department, site, spending thresholds, and approval levels. Power BI provides reporting on cycle times, spending trends, and governance metrics. The architecture supports configurable thresholds, automatic routing, escalations, audit tracking, and reporting.
